## Golden rules (read these)

1. ⚠️ **Money & access are real.** Recording a payment, accepting a proof, granting/revoking access, and sending SMS take effect immediately for real students. Check names and amounts.
2. ⚠️ **Some actions can't be undone.** Deleting a webinar also deletes its registrations. Prefer `Disable` over `Delete` when available. There is **no refund button** — escalate refunds.
3. **Missing buttons = permissions, not bugs.** Your role controls what you see. See [Roles & Permissions](roles-permissions).
4. **Two login codes exist** — the `Access` code (`NS-…`, LMS) and the 7-character `Portal` code (student portal). Most students need the `Portal` one.
5. **Exports contain personal data** — keep CSVs private.

## Quick map: where do I go for…?

| I want to… | Page |
|---|---|
| Follow up with an enquiry | `Sales` → `Lead CRM` |
| Check or verify a payment | `People` → `Payments & Finance` |
| Enroll a student / record offline payment | `People` → `Students & Enrollments` |
| Give or remove course access | Student profile / `People` → `Access at Risk` |
| Resend a login code | Student profile → copy `Portal` chip or `SMS` |
| See webinar sign-ups | `Academics` → `Webinars & Events` → `View registrants` |
| Send SMS | `Sales` → `SMS Mission Control` |
| Find no-shows / re-engage | `Overview` → `Business Analytics` → `Re-engagement segments` |
| Add/edit course content | `Academics` → `Content / LMS` |
| Add a staff login | `People` → `Staff & Roles` |
