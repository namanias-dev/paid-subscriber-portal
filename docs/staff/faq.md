# Common Tasks / FAQ

The everyday "how do I…" recipes, each verified against the real screens. Click-by-click.

## A student paid OFFLINE (cash/UPI) — enroll them and give access
1. `People` → `Students & Enrollments`. Search their phone.
2. Not there? Click **`+ Add Student`**, fill `Full name *` and `Phone (10-digit) *`. (You can also enroll + record payment right in this form.)
3. Open the profile → **`Enrolled courses`** → **`Enroll`** → pick the course and `Pay in full`/`EMI` → **`Enroll`**.
4. On the course card → **`Record payment`** → choose what to `Settle` → `Method` = `Cash`/`Bank Transfer`/`Offline UPI` → set `Date (IST)` → **`Record {amount}`**.
5. Share their **`Portal`** login code (copy the chip, or `SMS` the `Login Code Resend` template).

## A student lost their login code
1. `People` → `Students & Enrollments` → open the student.
2. Copy the **`Portal`** chip in the header (that's their portal code), or click **`SMS`** → choose **`Login Code Resend`** → **`Send`**.
> You can't regenerate a student's code; just resend the existing one.

## Verify a payment that's stuck as pending
1. `People` → `Payments & Finance`.
2. Find the row → click the small **`↻`** icon (`Re-verify this payment with ICICI`). Or use **`↻ Re-verify payments`** for all stuck ones.
3. The status updates to `PAID`, `FAILED`, or `ABANDONED`.

## A student says they paid but it shows unpaid (no online record)
1. Ask them to upload proof from their portal (`Report a payment` → `Submit proof`).
2. In `Payments & Finance`, use the `Proof uploaded` filter → open the proof → `View (signed) →` to check it.
3. If genuine, click **`✓ Accept payment & grant access`** and confirm. ⚠️ This grants access.

## Give a student free access to a course
Profile → **`Enroll`** → choose **`Complimentary`** → **`Enroll`**. Instant ₹0 access.

## Recover a student whose lectures got blocked (dues)
`People` → `Access at Risk` → find them → **`+1 month`** (or **`Lifetime`**).

## Register someone for a webinar by hand
Profile → **`Webinars registered`** → **`Register`** → choose webinar, amount (`0` = free) → **`Register`**.

## Send a reminder SMS to one person
On their Lead/Payment/Student/Registrant row, click **`SMS`** → pick an Approved/Active template → `Send`. (Needs the send-SMS permission.)

## Send an SMS to a group
`Sales` → `SMS Mission Control` → `Send SMS` → pick template → pick `Audience` → `Preview` → `Send to {count}`. ⚠️ Bulk and `Everyone` are Super-Admin only.

## Find genuine webinar attendees / no-shows
Don't rely on the `Attended` column. Use `Sales` → `SMS Mission Control` → `Send SMS` → audience `Attended` / `No-show`, or `Business Analytics` → `Re-engagement segments` → `Paid webinar · no Zoom click` (with CSV export).

## Add a new lead and start following up
`Sales` → `Lead CRM` → **`+ Add Lead`** → fill name + phone → **`Add Lead`**. Open it → set `Pipeline stage`, add a note with `Log`, contact via `WhatsApp`/`Call`/`SMS`.

## Export a contact list for calling
- Leads: `Lead CRM` → **`⬇ Export CSV`**.
- Payments: `Payments & Finance` → **`⬇ Export`** / **`⬇ Export (filtered)`**.
- Segments: `Business Analytics` → `Re-engagement segments` → **`Export CSV`**.
> ⚠️ All of these contain personal data — handle securely.

## Create a webinar and share it
`Academics` → `Webinars & Events` → **`+ New Webinar`** → fill Basic Details + Pricing → **`Create Webinar`**. Then **`Copy link`** / **`WhatsApp`** to share.

## Reset a teammate's password (Super Admin / staff manager)
`People` → `Staff & Roles` → `Manage` the person → **`Reset password`** → share the new temporary password once.
